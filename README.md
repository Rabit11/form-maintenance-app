# 表单维护 APP

独立运行的表单维护工具（与科研项目信息化管理平台无关）。

下载或克隆后，可放置在任意本地目录运行。

## 启动入口

- 本地首页：`http://127.0.0.1:18088/`
- 内网部署：`http://10.90.111.114:8086/`
- 兼容旧地址：`http://127.0.0.1:18088/form-maintenance-app`
- 后端接口：`/api/transition-tool`

## 数据库备份路径

按《数据库备份与 Changelog 存档方案》Phase A：SQLite 在线备份，与业务 data 卷分离。

| 环境 | 备份目录 |
| --- | --- |
| 本地开发 | `<项目目录>\server\backups` |
| Docker 容器内 | `/app/server/backups` |
| 远程宿主机（114） | `/data_SSD_21T/users/yanghuiran/yanghuiran/form-maintenance-app/backups` |

触发：确认入库 / 整表替换前、加载样本前、每日日备、接口 `POST /api/admin/backups`（总部/超管）。清单文件：`MANIFEST.jsonl`；每份备份旁路 `.sha256`。

## 首次运行

如果本文件夹没有 `node_modules`，先安装依赖：

```bat
npm.cmd install
npm.cmd install --prefix server
npm.cmd install --prefix web
```

启动（托管已构建的 `web/dist`）：

```bat
start-form-maintenance-18088.cmd
```

若修改了前端源码，需先重新构建：

```bat
npm.cmd run build --prefix web
```

## 双向级联配置（可复现）

权威文件（改完后请同步两份）：

- `doc/cascade-level-channel-office-type.json`
- `server/config/cascade-level-channel-office-type.json`

方案说明：`doc/层级渠道司局项目类型-双向级联方案.md`

级联维度：`层级 → 渠道(C) → 司局/处室(辅助) → 项目类型(D)`，支持双向回填；司局不写 Excel。

## 已固化的模板规则

- 总表模板：`预先研究项目信息-表头 (1).xlsx`
- 总表 Sheet：`预先研究项目信息`
- 导入导出范围：`A:AN`
- 总列数：`40`
- 数据起始行：第 `6` 行
- 拆分字段：`D列 项目类型`
- 项目类型字典来源：`sheet4!$A:$A`
- 项目来源/渠道字段：`C列 项目来源/渠道`
- 项目来源/渠道字典来源：`sheet4!$F$1:$F$5`

## 当前功能

- 角色登录（总部 / 项目类型主管 / 只读）
- 按 `项目类型` 拆分专项分表并授权给项目类型主管
- 主管登录后仅可维护本人负责的分表（在线编辑或上传分表）
- 总部上传总表预校验、拆分导出、整表替换
- 表头、字段、项目唯一性、经费逻辑预校验
- 上传人确认后再正式入库
- 记录批次、上传人、确认人、更新时间和字段变更明细（留痕）
- 导出最新总表 / 本人分表
- 总部导出总表 + 专项分表 ZIP 包
