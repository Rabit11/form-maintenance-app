# 层级 · 渠道(C列) · 司局/处室 · 项目类型(D列) 双向级联方案

> 适用：表单维护 APP（预先研究项目信息总表/专项分表）  
> 约束：**不改 Excel 列结构**；司局/处室仅筛选与编辑辅助；字段字典冲突服从现有 APP（sheet4 + 表头）  
> 依据：需求 V19「项目分类体系」+ 现有 `TRANSITION_LEVEL_SOURCE_OFFICE_TYPES` 映射

---

## 1. 目标

将四个维度做成**同一张合法组合表**上的双向联动：

| 序号 | 维度 | 落点 | 是否进 Excel |
| --- | --- | --- | --- |
| L1 | 层级 | B 列「级别」 | 是 |
| L2 | 渠道 | C 列「项目来源/渠道」 | 是 |
| L3 | 司局/处室 | 运行时字段 `orgOffice` | **否**（辅助） |
| L4 | 项目类型 | D 列「项目类型」 | 是 |

**双向级联**含义：

1. **下行（正向）**：选上层 → 收窄下层可选集合；非法已选值自动清空或改写。  
2. **上行（反向）**：选下层（尤其项目类型）→ **唯一路径时自动回填**上层；多路径时仅收窄上层选项。  
3. **横向一致性**：任意时刻四个字段的组合必须落在「合法路径表」内，否则提示待修正。

筛选栏与编辑弹窗共用同一套级联引擎（同一配置源）。

---

## 2. 设计原则

| # | 原则 | 说明 |
| --- | --- | --- |
| P1 | 单一配置源 | 合法路径只维护一处：`层级 → 渠道 → 司局 → [项目类型…]` |
| P2 | Excel 契约不变 | 导出/导入仍只认 B/C/D；`orgOffice` 不进 A:AN |
| P3 | APP 字典优先 | C/D 用词以现有表头 & sheet4 为准；V19 仅提供司局细分骨架 |
| P4 | 公司级司局 = 内部处室 | 科研项目处 / 科技发展处 / 技术基础处 |
| P5 | 可反推 | 历史行无 `orgOffice` 时，按 D 列反推默认司局 |
| P6 | 筛选宽松、编辑严格 | 筛选允许「只选层级」；编辑保存时 B/C/D 必填且路径合法 |

## 4. 数据模型

### 4.1 路径元组

```text
Path = { level, sourceChannel, orgOffice, projectType }
```

### 4.2 派生索引（由配置表自动生成，禁止手工再维护一份）

| 索引名 | 用途 |
| --- | --- |
| `sourcesByLevel[L]` | 正向：层级 → 渠道列表 |
| `officesByLevelSource[L][C]` | 正向：层级+渠道 → 司局列表 |
| `typesByLevelSourceOffice[L][C][O]` | 正向：三级 → 类型列表 |
| `typesByLevelSource[L][C]` | 兼容：未选司局时的类型并集 |
| `pathByType[D]` | 反向：类型 → 唯一 `{L,C,O}`（类型全局唯一则满回填） |
| `pathsByOffice[O]` / `pathsByChannel[C]` | 反向：多路径候选，用于收窄上层选项 |

### 4.3 行上字段

| 字段 | 存储 | 写入 Excel |
| --- | --- | --- |
| `level` | 行 JSON / B 列 | 是 |
| `sourceChannel` | 行 JSON / C 列 | 是 |
| `projectType` | 行 JSON / D 列 | 是 |
| `orgOffice` | 仅行 JSON | **否** |

导入 Excel 后：`orgOffice = pathByType[D].orgOffice`（有映射则反填；无映射则为空，筛选可见「未归类」策略见 §7）。

## 5. 双向联动手册

约定四个控件代号：`Lv` `Ch` `Of` `Ty`。  
操作时先计算「合法候选路径集合 `S`」，再投影到各下拉选项。

### 5.1 核心算法（筛选 / 编辑共用）

```text
输入：当前部分选择 Partial = { Lv?, Ch?, Of?, Ty? }
S0 = 全量合法路径表
S  = S0 中满足所有「已选非空字段」的路径子集

选项：
  LvOpts = unique(S.level)           // 若 Lv 未选；若已选则仍可展示全集或仅 S——见 5.3
  ChOpts = unique(S.sourceChannel)
  OfOpts = unique(S.orgOffice)
  TyOpts = unique(S.projectType)

回填规则（仅编辑态默认开启；筛选态可选）：
  若 |unique(S.level)| = 1 → 自动写入 Lv
  若 |unique(S.sourceChannel)| = 1 → 自动写入 Ch
  若 |unique(S.orgOffice)| = 1 → 自动写入 Of
  若 |unique(S.projectType)| = 1 → 自动写入 Ty

清空规则：
  若当前已选值 ∉ 对应 Opts → 清空该字段（及按策略清空下游，见 5.2）
```

### 5.2 正向（下行）— 改上层清空下层

| 用户操作 | 行为 |
| --- | --- |
| 改 `Lv` | 重算；`Ch/Of/Ty` 若非法则清空；选项按新 `S` |
| 改 `Ch` | 重算；`Of/Ty` 非法则清空 |
| 改 `Of` | 重算；`Ty` 非法则清空 |
| 改 `Ty`（且未开反向回填） | 仅校验是否仍在 `S` |

**未选司局时**：`TyOpts = typesByLevelSource[L][C]`（该渠道下全部类型），保证兼容旧用法。

### 5.3 反向（上行）— 改下层回填 / 收窄上层

| 用户操作 | 行为 |
| --- | --- |
| 选 `Ty` | 查 `pathByType[Ty]`：若唯一 → **强制回填** `Lv/Ch/Of`；同时锁定选项为该路径 |
| 选 `Of`（未选 Ty） | `S` 收窄；若层级/渠道唯一则回填；`TyOpts` 收窄为该司局下类型 |
| 选 `Ch`（未选 Lv） | 若该渠道只属于一个层级（如市科委→地方级、ZGSF→公司级）→ 回填 `Lv`；否则只收窄 |
| 选 `Lv` | 仅正向收窄（层级不能由司局单独唯一推出时不强行改） |

**多路径歧义**（现阶段应避免）：若未来一个项目类型挂在多个司局，反向选类型时：

1. 不自动回填 `Of`；  
2. `OfOpts` 展示候选司局；  
3. 保存前必须用户点选司局（仅编辑）。

当前配置保证 **项目类型 → 路径唯一**，反向可满回填。

### 5.4 筛选态 vs 编辑态差异

| 能力 | 筛选栏 | 编辑弹窗 |
| --- | --- | --- |
| 四字段均可空 | 是（「全部…」） | `Lv/Ch/Ty` 保存必填；`Of` 建议必填但允许空（空则保存时按 `Ty` 反填） |
| 选 Ty 反向回填 Lv/Ch/Of | 建议开启（减少操作） | **必须开启** |
| 改 Lv 清空下游 | 是 | 是 |
| 合法路径外组合 | 不出现在选项中 | 保存校验拦截 |

### 5.5 交互示意

```mermaid
flowchart LR
  Lv[层级 Lv] -->|下行收窄| Ch[渠道 Ch]
  Ch -->|下行收窄| Of[司局 Of]
  Of -->|下行收窄| Ty[类型 Ty]
  Ty -->|上行唯一回填| Of
  Of -->|上行唯一回填| Ch
  Ch -->|上行唯一回填| Lv
```

示例：

1. 只选 `Ty=MJKY` → 自动 `Lv=国家级, Ch=工信部, Of=装备二司`  
2. 先选 `Lv=国家级, Ch=工信部` → `Of∈{装备二司,装备一司,高新技术司}`，`Ty` 为三者并集  
3. 再选 `Of=装备二司` → `Ty` 仅 `MJKY`；若再满足唯一则可自动填 `Ty`  
4. 公司级选 `Of=科技发展处` → `Ty` 仅科技周 + 各大飞机研究院校；反向选某研究院校 → 回填 ZGSF + 科技发展处 + 公司级

## 6. 界面落地

### 6.1 筛选栏顺序（固定）

```text
[搜索] [全部层级] [全部渠道] [全部司局/处室] [全部项目类型] …
```

- 文案：司局旁 tooltip「辅助筛选，不写入 Excel」  
- 公司级提示：司局选项为内部处室名称  

### 6.2 编辑弹窗顺序（固定）

```text
项目级别* → 项目来源/渠道* → 司局/处室 → 项目类型*
```

- 司局说明：「辅助字段，不导出 Excel；保存时随行 JSON 留痕」  
- 修改四者任一，走 §5 引擎；保存写 B/C/D + `orgOffice`

### 6.3 选项禁用策略（可选增强）

- **硬清除**（现状推荐）：非法值直接清空，简单可靠。  
- **软禁用**：保留显示但 disabled + 灰显，适合审计对比——本阶段不做。

## 7. 历史数据与校验

| 场景 | 处理 |
| --- | --- |
| 有 D、无 orgOffice | 用 `pathByType` 反填展示与筛选；不强制回写 DB，直至下次保存 |
| D 不在映射表 | `orgOffice=''`；编辑保存时 warning：「项目类型未配置司局路径」；筛选「未归类」：`Of=''` 且选「全部司局」时仍可见 |
| C 与 D 冲突（如 C=工信部 但 D=预研三年） | 编辑保存 400；导入预校验 warning；以用户确认后的合法组合为准 |
| 层级立项后原则上不变更（V19） | 编辑若改 `Lv` 且已有历史批次：warning 提示「层级原则上不变更，需走数据变更」——**软提示不拦保存**（过渡工具） |

## 9. API / 前端契约（细化）

### 9.1 `GET /api/transition-tool` → `cascade`

```json
{
  "levels": ["国家级", "地方级", "公司级"],
  "sourcesByLevel": { "国家级": ["工信部", "科技部", "发改委"] },
  "officesByLevelSource": { "国家级": { "工信部": ["装备二司", "装备一司", "高新技术司"] } },
  "typesByLevelSourceOffice": { "国家级": { "工信部": { "装备二司": ["MJKY"] } } },
  "typesByLevelSource": { "国家级": { "工信部": ["MJKY", "国家科技重大专项--04专项", "1025"] } },
  "officeByType": { "MJKY": { "level": "国家级", "source": "工信部", "office": "装备二司" } },
  "pathByType": { "MJKY": { "level": "国家级", "sourceChannel": "工信部", "orgOffice": "装备二司", "projectType": "MJKY" } }
}
```

> `officeByType` 已存在；补齐对称别名 `pathByType`（与前者等价，便于双向引擎命名统一）。

### 9.2 前端级联引擎（建议抽模块）

`web/src/lib/cascadePath.ts`（待实现增强）：

```ts
type PartialPath = {
  level?: string
  sourceChannel?: string
  orgOffice?: string
  projectType?: string
}

type CascadeIndex = TransitionCascade // 含上表各索引

function resolveCascade(
  index: CascadeIndex,
  partial: PartialPath,
  mode: 'filter' | 'edit'
): {
  next: PartialPath          // 含自动回填后的值
  options: {
    levels: string[]
    sources: string[]
    offices: string[]
    types: string[]
  }
  valid: boolean
  issues: string[]
}
```

**双向开关**（编辑默认全开）：

```ts
{ forwardClear: true, reverseBackfill: true }
```

筛选栏：`reverseBackfill: true` 推荐开启（选类型后自动带出渠道/司局，减少误筛）。

### 9.3 导出查询参数

已支持：`level` `channel` `office` `projectType`  
双向仅影响 UI 状态拼装；导出过滤仍用最终四个参数。
